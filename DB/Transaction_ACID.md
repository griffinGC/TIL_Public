## ACID

1. 원자성 (Atomicity)
   - 작업중에 하나라도 생기면 rollback, 모두 성공했을 경우에만 commit 하는 것
   - 모두 성공 또는 실패
   - 부분적 갱신으로 발생할 수 있는 문제 방지
2. 일관성 (Consistency)
   - 작업 처리 결과가 항상 일관성 있어야 하는 것
   - 진행되는 동안 데이터가 변경되더라도 업데이트 된 데이터로 트랜잭션이 진행되는 것이 아니라, 처음에 트랜잭션을 진행하기 위해 참조한 데이터로 진행 되는 것
   - 각 사용자는 일관성 있는 데이터 볼 수 있음
3. 독립성 (Isolation)
   - 둘 이상의 트랜잭션이 동시에 병행 실행되는 경우 어느 하나의 트랜잭션이라도 다른 트랜잭션의 연산에 끼어들 수 없음
   - 하나의 트랜잭션이 완료될때까지, 다른 트랜잭션이 특정 트랜잭션의 결과 참조 불가
4. 지속성 (Durability)
   - 트랜잭션이 성공적으로 완료되었을 경우, 결과는 영구적으로 반영되는 것
